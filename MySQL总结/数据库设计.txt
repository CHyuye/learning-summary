数据类型
  ·使用数据类型的原则：够用就行，尽量使用取值范围小的，而不用大的，节省存储空间
  
 ·常用数据类型：
  整数： int，bit
  小数：decimal
  字符串：varchar, char  (varcahr——指定字符串的大小，char——当字符串未达到指定大小时，会用空格代替，所以建议用varchar)
  日期时间：date（2020-01-01），time（12:29:59），datetime（2020-01-10 12:29:59）
  枚举类型：（enum）——枚举类型指的是在enum后面指定给定的参数，用户只能在指定范围内选择

特殊说明的类型
  decimal表示浮点数，如 decimal(5, 2)表示共存有5位数，小数占两位；
  char表示固定长度的字符串，如char(3)，如果输入为‘ab’时，就会自动补一个空格‘ab ’；
  varchar为可变长度字符串
  字符串text表示存储大文本，字符大于4000时推荐使用
  对于图片，音频，视频等文件，不存储在数据库中，而是上传到服务器，然后在表中存储这个文件的保存路径

约束
 · 主键primary key：物理上存储的顺序
 · 非空not null：此字段不允许填写空值
 ·唯一unique：此字段不允许重复
 ·默认default：当不填写此值时使用默认值，如果填写以填写为准
 ·外键foreign key：对关键字段进行约束，当关系字段填写值时，会到关系的表中查询此值是否存在，如果存在成功，如果没有就报错
 
示例：
create table goods(
	id int unsigned primary key auto_increament not null,   -- ini类型，取正值，主键，自动增长，不能为空
	name varchar(30) not null,
	cate_name varchar(30) not null,
	brand_name varchar(40) not null,
	price decimal(10,3) not null default 0,  -- 浮点数类型取5位数，小数两位
	is_show bit not null default 1,  -- 是否展示，bit类型，不为空，默认展示 0代表false，1代表true
	is_saleoff bit not null default 0  -- 末尾不能再加逗号 默认不
)
 
数据库的设计
--三范式（Normal Form）
第一范式：强调的是列的原子性，即列不能够再分成其他几列。

第二范式：首先必须是范式一，另外包含两部分内容，一是表必须有主键；
二是没有包含在主键中的列必须完全依赖于主键，而不能只依赖于逐渐的一部分。

第三范式:首先是范式二，另外非主键必须依赖于主键，不能存在传递依赖，
即不能存在：非主键列A依赖于非主键列B，非主键列B依赖于主键的情况。

E-R模型
·E表示entry，实体，设计就像定义一个类一样，指定从哪些方面描述对象，一个实体转换成数据库中的一个表
·R表示relationship，关系，关系描述这两个实体之间的对应规则，关系的类型包含一对一，一对多，多对多


表设计规范
  1·单实例表数目必须小于500
  2.单表列数目必须小于50
  3.表必须有主键
    解读：主键递增，数据行写入可以提高插入性能，可以避免page分裂，减少表碎片提升空间和内存的空间
  4.禁止使用外键
    解读：外键会导致表与表之间耦合，update和delete操作都会涉及相关联的表，十分影响sql的性能，甚至会造成死锁，
高并发情况下容易造成数据库性能，大数据高并发业务场景数据库使用以性能优先

字段设计规范
  1.必须把字段定义为not null 并且提供默认值
    解读：null的列使索引/索引统计/值比较更加复杂，对MYSQL来说更难优化
    null这种类型mysql数据库需要对它进行特殊处理，增加数据库处理记录的难度，降低处理性能
    null值需要更多地存储空间
    对null的处理时候，只能采用is null或is not null 而不能采用=，in，<, <>,!=，not in等操作符号

禁止使用text，blob类型
   解读：会占用大量内存，浪费更多磁盘和内存空间，非必要的大量的大字段查询会淘汰掉热数据，导致内存命中率急剧降低

禁止使用小数存储货币
  解读：使用整数吧，小数容易导致钱对不上

必须使用varchar（20）存储手机号

如何取消外键约束？
  --先获取外间的名称
show create table goods;
  --获取后根据名称删除外键约束
alter table goods drop foreign ker 外键名称;

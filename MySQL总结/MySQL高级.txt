1 —— 视图
 视图是什么？
 1、视图就是一条select语句执行后返回的结果集
  2、视图是对若干张基本表的引用，一张虚表，查询语句执行后的结果，不存储具体数据（基本表数据变了，视图也会变）
 
作用：当你需要对多个数据表进行修改，为了保证查询出来的数据与之前相同，可以定义视图分析
  方便操作，特别是查询操作，减少复杂的sql语句，增强可读性

定义视图
  create view 视图名称 as select 语句;

查看视图
   show tables;


2 —— 事务
  所谓事务，它是一个操作序列，这些要么执行，要么不执行，它是一个不可分割的工作单位

事务的四大特性（ACID）
1、原子性（Atomicity）
   一个事务被视为一个不可分割的最小工作单位，整个事务中的所有操作要么全部成功提交，要么全部失败回滚，
对于一个事务来说，不可能只执行一部分操作，这就是事务的原子性

2、一致性（Consistency）
  数据库总是从一个一致性的状态到另一个一致性的状态。

3、隔离性（Isolation）
  通常来说，一个事务所做的修改在最终提交前，对其他事务是不可见的

4、持久性（durability）
  一旦事务提交，则所做的修改会永久保存到数据库

开启事务
begin或者start transaction

提交事务
commit

回滚事务
rollback

3 —— 索引
 索引的目的 
  索引的目的在于提高查询的效率，可以类比字典
 
索引的原理
   通过不断缩小想要获得数据的范围来筛选最终想要的结果，同时把随机的事件变成顺序事件。比如生活中的图书目录
火车站的车次表

索引的使用
  查看索引
show index from 表名;
  
  创建索引 —— 如果指定字段的字符串，需要指定长度，建议长度与定义字段时的长度一致，字段类型如果不是字符串，可以不填写长度部分
create index 索引名称 on 表名(字段长度);

删除字段
drop index 索引名称 on表名;
